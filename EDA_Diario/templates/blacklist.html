Como deve ser o ficheiro CSV

A primeira linha é sempre o cabeçalho, com os mesmos nomes de coluna que na tabela MySQL blacklist usa para inserções. O destino dos dados é a base configurada em EDA_MYSQL_DATABASE (ex.: plataforma_central).

  • tipo — obrigatório: um de CPF, NOME, TELEFONE, EMAIL (maiúsculas recomendadas).
  • valor — obrigatório: texto a bloquear (CPF só dígitos ou com pontuação; será normalizado no fluxo).
  • motivo — opcional: texto livre.
  • ativo — opcional: 1 / true / vazio importa; 0 / false ignora essa linha.

Colunas id e data_inclusao, se vierem dum export SQL, são ignoradas. Outras colunas extra também são ignoradas (apenas aviso após importar). Separador , ou ; é detetado automaticamente. Ficheiros UTF-8 ou com BOM são aceites.

Exemplo

tipo,valor,motivo,ativo
EMAIL,contacto.livre@gmail.com,Solicitou remoção,1
CPF,12345678901,,1
NOME,JOSÉ SILVA EXPORTADO,Blacklist operacional,
TELEFONE,11988776655,Apenas este número bloqueado,1

Linhas com o mesmo par tipo + valor fazem atualização (reactiva e mantém/atualiza motivo conforme envio).
